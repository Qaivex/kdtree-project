# kdtree

Реализация по теме «Пространственные структуры данных (k-мерное дерево
поиска)».

Проект содержит библиотеку k-мерного дерева поиска (k-d tree) на C11 и две
кластеризации поверх неё - Fuzzy C-means и DBSCAN. Поверх библиотеки
собрана консольная утилита `kdtree`, которая принимает CSV с
координатами точек, выполняет указанную операцию и выводит результат.
Для операций кластеризации в стандартный вывод также печатается
визуализация точек, раскрашенных по кластерам.

## Требования и сборка

Требования:
- компилятор C с поддержкой стандарта C11 (`gcc` или `clang`);
- `make`;
- стандартная библиотека C и `libm`.

Внешних зависимостей нет.

Релизная сборка:
```bash
make
```
В результате формируется исполняемый файл `build/kdtree`.

Отладочная сборка с AddressSanitizer и UBSan:
```bash
make debug
```
Формируется `build/kdtree_dbg`.

Очистка артефактов сборки:
```bash
make clean
```

Сборка вручную, без `make`:
```bash
cc -std=c11 -O2 -Iinclude src/*.c -lm -o kdtree
```

## Запуск

Общий вид:
```
./build/kdtree <csv-файл> <операция> [аргументы операции] [опции]
```

Примеры из ТЗ:
```bash
./build/kdtree data/lidar.csv -kd_nearest 1.0,2.0
./build/kdtree data/blobs.csv -cmeans 3
./build/kdtree data/lidar.csv -dbscan 0.5,5
```

Полный прогон по всем алгоритмам на тестовых датасетах:
```bash
make demo
```

## Формат входных данных

CSV-файл, одна точка на строку, координаты разделены запятыми:
```
# x,y
0.1234,2.7180
-1.0000,3.5000
```

Особенности:
- строки, начинающиеся с `#`, считаются комментариями и пропускаются;
- размерность определяется по первой непустой строке;
- поддерживаются размерности от 2 до 8.

## Операции CLI

| Операция | Аргументы | Назначение |
|---|---|---|
| `-kd_insert`    | `x,y[,z]`   | Построить дерево из файла, вставить точку, вывести размер |
| `-kd_nearest`   | `x,y[,z]`   | Поиск ближайшего соседа |
| `-cmeans`       | `c`         | Fuzzy C-means с `c` кластерами |
| `-cmeans_elbow` | `c_max`     | Метод локтя на диапазоне `c = 2..c_max` |
| `-dbscan`       | `eps,minPts`| DBSCAN с радиусом `eps` и порогом плотности `minPts` |

Опции:

| Опция | Назначение |
|---|---|
| `--no-ansi` | не выводить ANSI-визуализацию в стандартный вывод |
| `--width N` | ширина визуализации в символах (по умолчанию 80) |
| `--height N`| высота визуализации в символах (по умолчанию 30) |

## Визуализация

После выполнения операций кластеризации (`-cmeans`, `-cmeans_elbow`,
`-dbscan`) в стандартный вывод печатается «карта» облака точек, в
которой каждая точка отображается символом `*`, окрашенным в цвет
своего кластера. Под картой выводится легенда со списком кластеров и
цветом шума.

Для раскраски используется ANSI 24-bit truecolor - последовательность
вида `\x1b[38;2;R;G;B m`. Цвета берутся из фиксированной палитры на 16
оттенков, объявленной в `src/viz.c`. Цвет точки определяется выражением
`palette[cluster_id mod 16]`. Точки с `cluster_id == -1` (шум DBSCAN)
отображаются нейтрально-серым.

Для корректного отображения требуется терминал, поддерживающий
truecolor: GNOME Terminal, Konsole, kitty, Alacritty, Windows Terminal
и большинство современных эмуляторов. Если терминал не поддерживает
ANSI-цвета или вывод цветов нежелателен, печать можно отключить опцией
`--no-ansi`.

Размер выводимой карты задаётся опциями `--width` и `--height`
(значения по умолчанию - 80 символов в ширину и 30 в высоту).

Пример запуска с визуализацией:
```bash
./build/kdtree data/blobs.csv -cmeans 3
./build/kdtree data/moons.csv -dbscan 0.25,5
```

## Описание модулей

### `point` - точка в k-мерном пространстве

Определяет тип `rs_point_t` с фиксированной максимальной размерностью
`RS_MAX_DIM = 8`. Координаты размещаются непосредственно в структуре,
что обеспечивает плотное расположение в памяти и улучшает локальность
обращений. Содержит базовые операции вычисления евклидовых расстояний,
копирования и печати.

### `kdtree` - k-мерное дерево поиска

Реализация по схеме Bentley:
- разбиение пространства по оси `depth % dim` на каждом уровне;
- сбалансированное построение через медиану - `O(n log n)`;
- вставка - рекурсивный спуск без перебалансировки;
- удаление - замена удаляемой точки минимумом из правого поддерева по
  той же оси (при отсутствии правого поддерева - переносом левого
  поддерева вправо);
- поиск ближайшего соседа - DFS с отсечением по разделяющей
  гиперплоскости;
- range-запрос - DFS с отсечением, применяется в DBSCAN.

### `cmeans` - Fuzzy C-means и метод локтя

Реализация по формулам Бездека (1981):
- центроиды: `v_j = Σ_i u_ij^m · x_i / Σ_i u_ij^m`;
- матрица принадлежности: `u_ij = 1 / Σ_k (d_ij / d_ik)^(2/(m-1))`;
- целевая функция: `J_m = Σ_i Σ_j u_ij^m · d_ij²`;
- критерий останова: `||U_new − U_old||_∞ < tol` или достижение
  максимального числа итераций.

Вырожденный случай (`d_ij = 0`) обрабатывается явным назначением
`u_ij = 1` для совпавшего кластера и `0` для всех остальных.

Метод локтя реализован как многократный прогон FCM для `c = 2..c_max`
с последующим выбором значения `c`, максимизирующего вторую разность
`J(c-1) − 2·J(c) + J(c+1)`.

### `dbscan` - DBSCAN

Реализация по статье Ester et al. (1996):
- три категории точек: ядро (core), граничная (border), шум (noise);
- последовательный обход непосещённых точек, для каждой выполняется
  ε-запрос окрестности через k-d дерево;
- расширение кластера через BFS по очереди соседей;
- точки шума получают `cluster_id = -1`.

Использование k-d дерева для ε-запросов снижает среднюю сложность
алгоритма с `O(n²)` до `O(n log n)`.

### `csv` - CSV-парсер

Реализован без внешних зависимостей. Поддерживает комментарии (`#`),
пустые строки, переменное количество пробелов.

### `viz` - визуализация

Реализует ANSI-визуализацию точек в стандартном выводе и палитру цветов
кластеров.

### `main` - CLI

Разбор аргументов командной строки, загрузка CSV, диспетчеризация
операций, вывод результатов.

## Структуры данных

| Структура | Файл | Поля и назначение |
|---|---|---|
| `rs_point_t`         | `point.h`  | `coord[RS_MAX_DIM]`, `dim`, `cluster_id`, `index` |
| `rs_kdnode_t`        | `kdtree.h` | `point`, `axis`, `left`, `right` |
| `rs_kdtree_t`        | `kdtree.h` | `root`, `dim`, `size` |
| `rs_cmeans_params_t` | `cmeans.h` | `c`, `m`, `max_iter`, `tol`, `seed` |
| `rs_cmeans_result_t` | `cmeans.h` | `n`, `c`, `dim`, `membership[n*c]`, `centroids[c]`, `objective`, `iterations` |
| `rs_dbscan_params_t` | `dbscan.h` | `eps`, `min_pts` |
| `rs_dbscan_result_t` | `dbscan.h` | `n`, `cluster_ids[n]`, `n_clusters`, `n_noise` |
| `rs_rgb_t`           | `viz.h`    | `r`, `g`, `b` - 24-битный цвет |

Инвариант k-d дерева: для любого узла все точки в левом поддереве
имеют `coord[axis] < node->point.coord[axis]`, в правом поддереве -
`≥ node->point.coord[axis]`.

## Список функций (API)

### k-d дерево (`include/kdtree.h`)

- `rs_kdtree_create(dim)` - создание пустого дерева.
- `rs_kdtree_destroy(tree)` - освобождение ресурсов.
- `rs_kdtree_build(points, n, dim)` - сбалансированное построение.
- `rs_kdtree_insert(tree, p)` - вставка точки.
- `rs_kdtree_remove(tree, p)` - удаление точки.
- `rs_kdtree_nearest(tree, q, out, &sqdist)` - поиск ближайшего соседа.
- `rs_kdtree_range(tree, q, r, out, cap)` - поиск точек в радиусе
  (используется внутри DBSCAN).
- `rs_kdtree_size(tree)` - число точек в дереве.

### Точка (`include/point.h`)

- `rs_point_sqdist(a, b)` - квадрат евклидова расстояния.
- `rs_point_dist(a, b)` - евклидово расстояние.
- `rs_point_copy(dst, src)` - копирование точки.
- `rs_point_print(p)` - печать точки.

### Fuzzy C-means (`include/cmeans.h`)

- `rs_cmeans_default_params(c)` - параметры по умолчанию
  (`m = 2`, `max_iter = 200`, `tol = 1e-5`, `seed = 42`).
- `rs_cmeans_run(points, n, dim, params)` - выполнить кластеризацию.
- `rs_cmeans_result_free(res)` - освобождение результата.
- `rs_cmeans_hard_assign(res, out_points)` - жёсткое назначение точек
  по `argmax u_ij`.
- `rs_cmeans_elbow(points, n, dim, c_max, out_J)` - метод локтя.

### DBSCAN (`include/dbscan.h`)

- `rs_dbscan_run(points, n, dim, params)` - кластеризация.
- `rs_dbscan_result_free(res)` - освобождение результата.

### CSV (`include/csv.h`)

- `rs_csv_load(path, &points, &n, &dim)` - загрузка CSV.

### Визуализация (`include/viz.h`)

- `rs_viz_color_for_cluster(id)` - получение цвета по `cluster_id`.
- `rs_viz_print_ansi(points, n, width, height)` - печать карты точек
  в стандартный вывод.
